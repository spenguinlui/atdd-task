# 批次匯入保險 CookieOverflow — BA 報告

> **Task ID**: b9f547d2-ae6b-453d-8597-4d3ba21701cd | **Type**: fix | **Project**: core_web | **Date**: 2026-04-10

## 需求摘要

在「批次建立保險」頁面透過上傳檔案匯入保險資料時，若檔案中有大量資料不符合格式要求，系統會因為錯誤訊息過多而無法正常顯示頁面，直接出現系統錯誤畫面。使用者無法得知哪些資料有問題，也無法繼續操作。

## 業務分析結論

- 問題發生在「批次建立保險」頁面的「匯入」功能
- 當上傳的檔案中有許多筆資料格式不正確時（例如案場名稱找不到、保險類別填寫錯誤、日期或金額為空），系統會產生大量的錯誤提示
- 這些錯誤提示的總量超過了瀏覽器的儲存限制，導致系統直接當掉
- 正常情況下，系統應該要能顯示錯誤摘要，告知使用者有哪些資料需要修正
- 修正後，即使檔案中有大量錯誤，頁面仍能正常顯示，並提供足夠的資訊讓使用者修正檔案

## 驗收條件

- 上傳含有大量格式錯誤的檔案時，頁面不再出現系統錯誤，能正常顯示錯誤提示
- 錯誤提示顯示前面幾筆具體的錯誤內容，並在末尾告知總共有多少筆錯誤
- 使用者能根據錯誤提示修正檔案後重新上傳
- 上傳格式正確的檔案時，匯入功能維持正常運作，不受此修正影響
