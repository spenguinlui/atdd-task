The core of ATDD task is accepting deliverables stage by stage: requirements → spec → tests → implementation → review. Each deliverable must clear its acceptance gate before the next step — layered checks that buy you stable, predictable development.
Hand a requirement to AI; you only confirm at key checkpoints — no technical knowledge needed.

Quick Start
  • Copy .env.example to .env; fill in project path and test account
  • Start your first task: /feature sf_project, Project approval workflow
  • The AI starts asking questions until it has 95% confidence in the requirements
  • Advance stage by stage: /continue; check progress anytime: /status
  • Not sure how to fill a variable? Ask the framework administrator

Task Types
  • /atdd {one-line requirement} — natural-language entry; AI picks the type
  • /feature — develop a new feature (full pipeline)
  • /fix — fix a bug (skips the spec stage)
  • /refactor — refactor (review enforces "behavior unchanged")
  • /epic — large feature; AI proposes a breakdown first

Workflow: what AI does, what you do
  • Clarify → spec → tests → implement → review → gatekeeper decision
  • AI runs most stages automatically; you only confirm at handoff points
  • Only six points need you: start task · answer requirements · E2E decision · confirm spec · choose fix scope · close after GO
  • You can review the previous stage's output before each /continue
  • AI won't start implementing until confidence reaches 95%

Outputs & Acceptance
  • Acceptance criteria (Given / When / Then) — confirm AI understood your requirement
  • Automated tests + reviewed, test-passing code changes
  • Manual verification guide: step-by-step checklist from the gatekeeper
  • How you know it's right: gatekeeper GO → tick the guide → /verify done
  • A NO-GO explains why — adjust as directed, then /continue

Limits & Troubleshooting
  • Won't do: touch production data, implement before requirements are confirmed, change what you didn't ask for
  • The vaguer the requirement, the more questions — think through key business rules first
  • E2E needs Chrome open and reachable to staging, with test data on the account
  • /continue does nothing → follow the AI's hint to complete the previous stage
  • Task stuck → /abort and restart, or check the operation manual
