## 解讀（對 atdd-task 的意義，不替業主決定 model）

1. **結構 rubric 飽和**：4 個 model 幾乎全 7/7 → 此 rubric 不能拉開強弱。真正區分在「陷阱覆蓋 + 信心度校準」。
2. **信心度校準是 harness 相關訊號**：atdd 有「信心度 ≥95% 才進 spec」硬閘門。gpt-5.5 自評最高（82–86%）但陷阱覆蓋最低（3/5）= 「自信但漏邊界」，對信心度閘門最危險（可能過度自信闖關）。Opus 最保守（62–72%）對這種「需求本就有歧義」的題目反而**校準較誠實**。
3. **Opus**：最精簡（197 行）卻抓滿 5 陷阱 + 最保守信心 = 效率 + 周全 + 不浮誇。
4. **Sonnet**：陷阱覆蓋與 Opus 同（5/5）、信心度中庸 = 性價比選項。
5. **Haiku**：4/5（漏孤兒反向）+ 一次風險沒用 id 格式 + 偏自信（78%）= 便宜階段可用、subtle 邊界較弱。

## 人工抽看驗證（2026-05-22，已複核 gpt-5.5 兩個「漏」）
- **孤兒反向 = 真漏**：gpt-5.5 三次的孤兒全是正向（對帳檔有、系統無，即需求原文），無一想到反向。Opus（R6 + S11-edge + A3 反向缺漏待確認）、Sonnet（信心度維度明列「反向孤兒完全未定義」+ S7-alt）**都主動標註此為需求未涵蓋的缺口**——資深訊號：質疑規格完整性。
- **解析變體 = 偏淺非全無**：gpt-5.5 有泛提「格式可辨識」+ 客戶編號格式，但未到 CRLF/千分位/引號 具體層級（具體變體計數 0）。Opus 3 次、Sonnet 5 次有點具體；Haiku 0。grep 的「·」對 gpt-5.5 略嚴。
- **民國曆 = 全體盲區**：12 次 matrix run 皆 0；但 Sonnet smoke 那次有 → **N=3 變異實證**（同 model 不同次會漂）。

## 誠實校準（別過度解讀）
- 陷阱偵測是**關鍵字近似**；上方已對兩個爭議點人工複核。
- N=3 樣本小，信心度數字會跑動。
- 比的是「平面提示寫 spec」，非完整 specist runtime。

## eval 設計待強化（下一版 gold）
- 第 2 項（風險≥3）數 R-id → Haiku 用非 R-id 格式時誤判；改「risk 區塊條目數」更穩。
- rubric 需加**能拉開差距**的題項：陷阱覆蓋自動化（語意比對非關鍵字）、信心度校準對照（自評 vs 實際歧義數）、mitigation 可追溯性。
